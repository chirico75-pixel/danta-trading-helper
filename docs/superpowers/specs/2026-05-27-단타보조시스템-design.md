# 단타 보조 시스템 설계 문서

**작성일:** 2026-05-27  
**상태:** 승인됨

---

## 1. 개요

단타 매매를 보조하는 로컬 웹 앱. Python FastAPI 백엔드가 KIS Open API와 연동해 실시간 데이터를 제공하고, 브라우저 단일 화면에서 4개 패널을 운용한다. 영웅문 HTS 옆에 나란히 띄워 사용하는 것을 전제로 설계한다.

---

## 2. 기술 스택

| 레이어 | 기술 |
|---|---|
| 백엔드 | Python 3.11+, FastAPI, uvicorn |
| 스케줄러 | APScheduler (REST 5초 폴링) |
| 실시간 | websockets (KIS WebSocket 구독) |
| 브라우저 푸시 | SSE (Server-Sent Events) |
| DB | SQLite + aiosqlite |
| 환경 변수 | python-dotenv |
| 프론트엔드 | 순수 HTML/CSS/JS 단일 파일 (외부 CDN 없음) |

---

## 3. 아키텍처

```
브라우저 (index.html)
  ├── SSE 수신 ←──────── FastAPI /events
  └── REST 호출 ←──────── FastAPI /api/*

FastAPI 백엔드
  ├── 폴링 스케줄러 (5초) ──→ KIS REST API  →  상승률 랭킹 갱신
  ├── WS 매니저 ────────────→ KIS WebSocket →  관심종목 실시간 체결
  ├── SSE 브로드캐스터 ─────→ 브라우저 이벤트 전송
  ├── 일지 API ────────────↔ SQLite (trades)
  └── 관심종목 API ─────────↔ SQLite (watchlist)
```

## 5. 핵심 기능 명세

### 5.1 실시간 급등 랭킹

- KIS REST `GET /uapi/domestic-stock/v1/ranking/fluctuation` 5초마다 호출
- 코스피 / 코스닥 / 전체 탭 전환 가능
- 등락률 20% 이상 종목 빨간색 하이라이트
- 거래량 배수(전일 대비) 함께 표시
- 종목 클릭 시 상한가 체크리스트 패널에 자동 입력

### 5.2 상한가 체크리스트

종목 코드 입력(또는 랭킹 클릭) 시 아래 항목 자동 점검 후 100점 만점 점수 산출:

| 항목 | 점수 | 판단 방법 |
|---|---|---|
| 당일 호재 뉴스 | +30 | 사용자 수동 체크 |
| 거래량 3배 이상 | +20 | KIS API 거래량 비교 |
| 등락률 20% 이상 | +20 | KIS API 현재가 |
| 네이버 실검 등재 | +15 | 사용자 수동 체크 |
| 시총 1,000억 이하 | +10 | KIS API 시가총액 |
| 연속 상한가 아님 | +5 | KIS API 전일 등락 |

- 70점 이상: 진입 검토 (초록)
- 50~69점: 관망 (노랑)
- 50점 미만: 제외 (빨강)
- 분석 이력 `checklist_history` 테이블에 저장

### 5.3 강일지

- 매수 / 매도 기록 폼: 종목명, 가격, 금액, 진입 이유(호재 뉴스)
- 매도 기록 시 같은 날 동일 종목 매수 건 중 가장 최근 건과 매칭해 손익 계산 (LIFO)
- 일별 총 손익 표시
- 최근 7일 손익 미니 바 차트 (inline SVG)
- 기록 수정 / 삭제 가능

### 5.4 관심종목 실시간 모니터링

- 종목 코드 입력 → watchlist 테이블 저장 → KIS WebSocket 구독 자동 전환
- 실시간 현재가, 등락률 표시
- 목표가 / 손절가 설정 → 해당 가격 도달 시 브라우저 알림 (Notification API)
- 관심종목 제거 시 WebSocket 구독 해제
- 동시 최대 20종목 (KIS WebSocket 제한 25개 중 5개 여유)

## 6. 데이터 모델

```sql
-- 매매 일지
CREATE TABLE trades (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    date       DATE    NOT NULL,
    ticker     TEXT    NOT NULL,
    name       TEXT    NOT NULL,
    side       TEXT    NOT NULL CHECK(side IN ('BUY','SELL')),
    price      INTEGER NOT NULL,
    amount     INTEGER NOT NULL,
    reason     TEXT,
    pnl        INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 관심종목
CREATE TABLE watchlist (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    ticker       TEXT    NOT NULL UNIQUE,
    name         TEXT    NOT NULL,
    target_price INTEGER,
    stop_loss    INTEGER,
    memo         TEXT,
    added_at     DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 체크리스트 이력
CREATE TABLE checklist_history (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    ticker     TEXT    NOT NULL,
    date       DATE    NOT NULL,
    score      INTEGER NOT NULL,
    news_ok    BOOLEAN DEFAULT 0,
    volume_ok  BOOLEAN DEFAULT 0,
    surge_ok   BOOLEAN DEFAULT 0,
    naver_ok   BOOLEAN DEFAULT 0,
    cap_ok     BOOLEAN DEFAULT 0,
    no_chain   BOOLEAN DEFAULT 0,
    result     TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 7. KIS API 연동

| 기능 | 엔드포인트 | 방식 |
|---|---|---|
| 상승률 랭킹 | `/uapi/domestic-stock/v1/ranking/fluctuation` | REST, 5초 폴링 |
| 종목 현재가 | `/uapi/domestic-stock/v1/quotations/inquire-price` | REST, 요청 시 |
| 시가총액 | `/uapi/domestic-stock/v1/quotations/inquire-price` | REST, 요청 시 |
| 실시간 체결 (실전) | `wss://ops.koreainvestment.com:21000` | WebSocket |
| 실시간 체결 (모의) | `wss://openapivts.koreainvestment.com:31000` | WebSocket |
| Access Token | `/oauth2/tokenP` | REST, 24시간 캐시 |

**토큰 관리:** 앱 시작 시 `.token_cache` 파일 확인 → 유효하면 재사용, 만료 시 재발급.

## 8. 에러 처리

| 상황 | 처리 방법 |
|---|---|
| KIS API 429 rate limit | 지수 백오프 재시도 (1s → 2s → 4s, 최대 3회) |
| WebSocket 연결 끊김 | 5초 후 자동 재연결, UI에 "재연결 중" 배지 표시 |
| 장 마감 (15:30 이후) | 폴링 스케줄러 자동 중지, 헤더에 "장 마감" 표시 |
| `.env` 키 미설정 | 앱 시작 시 검증 → 설정 방법 안내 후 종료 |
| DB 쓰기 실패 | 에러 로그 기록, UI에 토스트 메시지 표시 |

## 9. 실행 방법

```bash
# 1. 의존성 설치
pip install -r requirements.txt

# 2. KIS API 키 설정
cp .env.example .env
# .env에 APP_KEY, APP_SECRET, ACCOUNT_NO 입력

# 3. 실행
python main.py

# 4. 브라우저에서 접속
# http://localhost:8000
```
